# Railway — лаба по монадам (Вариант 13)

## Запуск

```bash
sbt run
```

Всё. Компилируется и стартует интерактивное меню прямо в терминале.

## Структура

```
src/main/scala/
├── Monads.scala   — Reader / Writer / State / IO + given-инстансы
├── Domain.scala   — Route, Train, Ticket, Config, World
├── Tariff.scala   — Reader: цены, багаж, штрафы
├── Explain.scala  — Writer: логи операций
├── Cash.scala     — State: бронь, возврат, новый день
└── Program.scala  — IO: меню, склейка монад
```

## Меню

```
=== Касса ===
 1) показать состояние
 2) купить билет
 3) вернуть билет
 4) добавить поезд
 5) следующий день
 0) выход
```

Старт: два поезда (`001` Москва→Питер 10 мест, `002` Москва→Нижний 8 мест), день 1.

## Вариант 13. Железнодорожная касса

Сюжет: касса продаёт билеты, учитывает багаж и свободные места.

### Reader

Из окружения:

- тарифы по направлениям;
- доплата за багаж;
- правило выбора места;
- штраф за возврат.

Реализовать:

- `ticketPrice(route, classType)`;
- `baggageCost(weight)`;
- `seatAvailable(train, seat)`;
- `refundAmount(ticket)`.

### Writer

Логировать:

- выбор маршрута;
- расчёт цены;
- назначение места;
- возврат.

### State

Состояние:

- места по поездам;
- проданные билеты;
- выручка.

Переходы:

- `bookTicket`;
- `cancelTicket`;
- `addTrain`;
- `nextDay`.

### IO

Сценарий:

- выбрать поезд;
- выбрать место;
- купить или вернуть билет.
